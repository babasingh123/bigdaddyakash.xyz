# Design 2048 Game

!!! note "The disguise"
    Interviewer says: **"Implement the slide-and-merge logic of 2048"**
    They mean: **"Reduce four-direction logic to one-direction logic via matrix rotation, then collapse each row"**

## Problem

Given a 4×4 board of integers (0 = empty), a `move(direction)` slides every tile toward that edge and merges equal adjacent tiles. Each tile can merge **at most once per move**.

## What it really tests

- Avoiding four near-identical implementations. Solve "slide left" once; for the other three directions, **rotate** the board, slide left, then rotate back.
- The collapse logic per row: drop zeros, merge adjacent equals (with a `merged` flag so `[2, 2, 2, 2]` becomes `[4, 4, 0, 0]` and not `[8, 0, 0, 0]`).
- Knowing matrix rotation tricks — `rotate_cw = list(zip(*board[::-1]))`.

## Approach

1. **Normalize**: rotate so the move direction becomes "left".
2. **Collapse each row**:
    - Filter zeros.
    - Walk left to right; if the previous tile equals current and hasn't merged yet, double it and skip current.
    - Pad with zeros back to row length.
3. **Denormalize**: rotate back.

Number of rotations per direction:

| Direction | Rotations CW before slide |
|-----------|---------------------------|
| Left  | 0 |
| Up    | 1 |
| Right | 2 |
| Down  | 3 |

To rotate back, apply `(4 - rots) % 4` more CW rotations.

## Code sketch

```python
class Game2048:
    def __init__(self, board: list[list[int]]):
        self.board = [row[:] for row in board]

    def _collapse_left(self, row):
        filtered = [x for x in row if x != 0]
        out, skip = [], False
        for i, v in enumerate(filtered):
            if skip:
                skip = False
                continue
            if i + 1 < len(filtered) and filtered[i + 1] == v:
                out.append(v * 2)
                skip = True
            else:
                out.append(v)
        return out + [0] * (len(row) - len(out))

    @staticmethod
    def _rot_cw(b):
        return [list(row) for row in zip(*b[::-1])]

    def move(self, direction: str) -> None:
        rots = {"L": 0, "U": 1, "R": 2, "D": 3}[direction]
        for _ in range(rots):
            self.board = self._rot_cw(self.board)
        self.board = [self._collapse_left(r) for r in self.board]
        for _ in range((4 - rots) % 4):
            self.board = self._rot_cw(self.board)
```

## Complexity

- `move`: O(n²) for an `n × n` board — rotation and collapse are linear in cell count.
- Space: O(n²).

## Edge cases

- A row of equal tiles `[2, 2, 2, 2]` → `[4, 4, 0, 0]`, **not** `[8, 0, 0, 0]` (the merged flag matters).
- A move that doesn't change the board: detect by comparing pre- and post-snapshots if you need to decide whether to spawn a new tile.
- Empty board: no-op.
